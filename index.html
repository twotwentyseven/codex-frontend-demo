CODEXFIT Component Demo

This a demo of all the CODEXFIT components, if you believe something has been missed please contact a member of the CODEXFIT team.

Documentation for components can be found here.

CODEXFIT Set up guide.

Highlight copy will need to be replaces with your CODEXFIT url

                    
    <script src="https://demo.codexfit.com/autoconfigure.js"></script>

This script will create the neccessary script, style and translation files as well as connecting the frontend with the codexfit backend.

CODEXFIT Styles

A unique css-variables file will need to be created for each project within this file you will determine all styles such as font-families, colours, padding and size.

                    
    <link rel="stylesheet" type="text/css" src="https://www.yourdomain.com/css-variables.css"></link>

For the full list go to the css-variable page.

Component initialisation

All component must be nested within an element with the data-attr of data-codex. Without this attr no codexfit component will render

                    
    <div data-codex>
        <codex-bundles></codex-bundles>
    </div>
